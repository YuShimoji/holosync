# SP-018: 動画追加UX改善

## 概要

動画追加の手間を削減し、「見たい動画をすぐに見始められる」体験を実現する。
3つの改善軸を段階的に実装する。

## Phase 1: ワンクリック再追加（履歴の活用強化）

### 現状の問題
- `history.js` に再生履歴（最大30件）が保存されているが、履歴UIは「見た記録」としてのみ機能
- 毎回同じチャンネルの配信を視聴するユーザーが、毎回URLをコピーして追加する必要がある

### 仕様

#### 1-A. 履歴からのワンクリック再追加
- 履歴リスト (`#watchHistoryList`) の各エントリに「再追加」ボタンを配置
- クリックで `createTile(videoId)` を即座に実行
- 既にタイルに存在する動画はボタンをグレーアウト（`hasVideo(id)` で判定）
- 追加後にトースト通知: 「{title} を追加しました」

#### 1-B. よく見るチャンネルのピン留め
- 履歴データから `channel` 名でグルーピングし、視聴頻度を算出
- 上位5チャンネルを「よく見るチャンネル」としてサイドバー上部に表示
- 各チャンネル名の横に「最新を検索」ボタン → チャンネルの最新動画一覧を表示
- データソース: 既存の `watchHistory` を集計（新規ストレージキー不要）

#### 1-C. セッション復元ショートカット
- 前回のセッション（タイル構成）を自動保存
- 起動時に「前回の構成を復元しますか?」バナーを表示
- ワンクリックで全タイルを復元
- データ: `lastSession` キーに `videoIds[]` を保存（既存の `persistVideos` と連携）

### 完了条件
- [x] 履歴リストの各エントリに「+」ボタンが表示される
- [x] ボタンクリックでタイルが追加され、トースト通知が表示される
- [x] 既存タイルと重複する場合はボタンがグレーアウトされる
- [x] よく見るチャンネル上位5件がサイドバーに表示される
- [x] 前回セッション復元バナーが起動時に表示される

### 実装詳細（Phase 1 完了）

- history.js: `renderWatchHistory` に「+」ボタン追加、`showToast` 連携、チャンネル集計・描画
- ui.js: `showToast(message, durationMs)` ユーティリティ追加
- player.js: `persistVideos` で `lastSession` を自動保存（非空時のみ）
- main.js: 起動時に `videos` 空 & `lastSession` 有の場合に復元バナー表示、チャンネル検索コールバック
- index.html: 復元バナー、よく見るチャンネルセクション追加
- styles/main.css: 各新UI要素のCSS追加

## Phase 2: 検索フローの高速化

### 現状の問題
- `searchbrowser.js` で検索→結果カード表示→1件ずつ追加の流れ
- 複数動画を追加したい場合、カードを1つずつクリックする必要がある
- `search.js` の検索履歴（5件）が `searchbrowser.js` と連携していない

### 仕様

#### 2-A. 複数選択＋一括追加
- 検索結果カードに選択チェックボックスを追加
- Ctrl+クリック or チェックボックスで複数選択
- 選択中は画面下部にフローティングバー表示: 「{N}件選択中 | すべて追加 | 選択解除」
- 「すべて追加」で選択した全動画を一括で `createTile`

#### 2-B. 検索履歴の統合
- `searchbrowser.js` の検索欄に `search.js` の検索履歴（`searchHistory` 最大5件）をドロップダウン表示
- 履歴クリックで即座に再検索
- `searchbrowser.js` での検索も `storageAdapter.saveSearchHistory()` に保存

#### 2-C. プレビュー強化
- 検索結果カードに動画の長さ（duration）と再生回数を表示
- サムネイルホバーで拡大プレビュー（320x180 → 480x360）
- ライブ配信のカードには赤い「LIVE」バッジを表示（既存の `.sb-live-badge` を活用）

### 完了条件

- [x] 検索結果カードが複数選択可能
- [x] フローティングバーで一括追加が機能する
- [x] 検索履歴ドロップダウンが表示・再検索可能
- [x] カードに duration / viewCount が表示される

### 実装詳細（Phase 2 完了）

- searchbrowser.js: チェックボックス追加、selectedVideoIds Set管理、セレクションバー動的生成
- searchbrowser.js: doSearch後にvideos.list API呼び出しでcontentDetails+statistics取得（1 unit/12件）
- searchbrowser.js: 検索入力フォーカス時に履歴ドロップダウン表示、クリックで再検索
- searchbrowser.js: showToast連携（個別追加・一括追加）
- styles/main.css: チェックボックス、セレクションバー、履歴ドロップダウン、再生時間バッジのCSS
- 仕様変更: サムネイルホバー拡大は見送り（サイドバー幅の制約で効果が薄い）。代わりにサムネイル上に再生時間バッジを追加

## Phase 3: スマート提案

### 現状の問題
- 「何を見るか」の判断がユーザーに委ねられている
- チャンネル監視はライブ配信のみ検出。アーカイブ動画は対象外
- 関連動画の提案がない

### 仕様

#### 3-A. チャンネル最新動画フィード
- 登録チャンネル (`channelWatchList`) の最新動画（ライブ以外も含む）を取得
- サイドバーに「おすすめ」セクションを新設
- 各チャンネルの最新3件をカード形式で表示
- API: `search.list` with `channelId` + `order=date`（ライブチェックと同じエンドポイント）
- 更新頻度: チャンネル監視のポーリングに相乗り（追加APIコスト最小化）

#### 3-B. 「一緒に見られている」提案
- 再生履歴から「同じセッションで一緒に追加された動画」のパターンを学習
- 1つの動画を追加した時に「この動画と一緒に見られています」を表示
- ローカルデータのみ（API不要）
- データ: 既存の `watchHistory` + セッション単位のグルーピング（`watchedAt` の近接で判定）

#### 3-C. クイック提案バー
- 動画追加セクションの上部に横スクロール可能な提案バーを配置
- 内容: チャンネル最新動画 + 一緒に見られた動画 + ライブ配信中
- 各提案はサムネイル + タイトルのコンパクト表示
- ワンタップで追加

### 完了条件
- [ ] 登録チャンネルの最新動画がカード表示される
- [ ] 「一緒に見られています」提案が表示される
- [ ] クイック提案バーが横スクロール可能に表示される

## 実装優先度

| Phase | 効果 | 実装コスト | API依存 | 推奨 |
|-------|------|-----------|---------|------|
| Phase 1 | 高（反復作業の削減） | 低（既存基盤活用） | なし | 最優先 |
| Phase 2 | 中（探索効率向上） | 低〜中 | 部分的 | Phase 1後 |
| Phase 3 | 高（判断コスト削減） | 中〜高 | 3-Aのみ | Phase 2後 |

## 影響範囲

- **変更ファイル**: history.js, searchbrowser.js, channel.js, input.js, main.js, index.html, styles/main.css
- **新規モジュール**: なし（既存モジュールの拡張で対応）
- **ストレージ追加**: `lastSession` キーのみ（Phase 1-C）
- **既存機能への影響**: なし（追加機能のみ）
